# 크립토 패턴 트레이딩 봇 — 테스트베드 (웹 UI)

기획 배경은 [기획서.md](기획서.md) 참고.

> 목적(v1.5): "**급등·급락 직전 구간(전조)에 공통 패턴이 있는가?**"를 리프트로 **안전하게(가짜로) 검증**.
> 실거래 없음. 데이터 조회에 API 키 불필요.

## 구조

```
[Next.js + TS UI]  브라우저에서 파라미터 조작 · 결과 시각화
      │  HTTP (파라미터 + 패턴별 오버라이드)
      ▼
[FastAPI]  api/server.py — 엔진을 API로 감쌈
      ▼
[Python 엔진]  engine/
   data.py       ① 바이낸스 분봉 받아오기 + parquet 캐시
   events.py     ② 급등/급락 라벨링 (트리플 배리어: K캔들 내 ±3×ATR)
   precursor.py  ③ 이벤트 직전 L캔들 클러스터링 → 전조 템플릿 + 리프트 평가
   patterns.py   (v1) Matrix Profile 반복 모양 발견 — 비교용으로 유지
   simulate.py   ④ TP/SL 청산 시뮬레이션 (ATR 배수 per-트레이드 지원)
   service.py    ⑤ 전체 조율 → UI용 JSON 반환 (mode: precursor | motif)
   main.py       (선택) 콘솔로 빠르게 확인하는 CLI
```

## 실행법 (터미널 1개)

```powershell
cd web
npm run dev       # 웹(3000) + 엔진 API(8000) 동시 실행, Ctrl+C로 둘 다 종료
```

그다음 브라우저에서 **http://localhost:3000** 접속.
왼쪽에서 파라미터를 정하고 **▶ 백테스트 실행**.

> 따로 실행하려면: `npm run web` / `npm run api` (또는 구버전 `./run-api.ps1`, `./run-web.ps1`)

> 콘솔로만 빠르게 보려면: `.venv/Scripts/python.exe -m engine.main`

## UI에서 할 수 있는 것

- **발견 방식 선택**: 급등·급락 **전조 발견(v1.5, 기본)** 또는 반복 모양(v1, 비교용)
- **이벤트 정의 조작**: 판정 구간 K / 이벤트 기준(×ATR) / 무효 기준(×ATR) — 단, 결과 보고 자꾸 바꾸면 과최적화
- **전조 발견 설정**: 전조 길이 L / 클러스터 수 / 최소 크기 / 거래량 채널 포함 여부
- **리프트 검증**: 패턴 카드에서 학습·검증 리프트(매칭 후 이벤트 확률 ÷ 기저율)와 `자동선별`·`검증 통과` 배지 확인
- **숏 지원**: 급락 전조는 숏(매도)으로 거래 (롱▲ / 숏▼)
- **ATR 기반 자동 익절/손절**: 이벤트 정의와 동일한 배수로 진입 시점마다 자동 계산 (카드에서 고정 %로 오버라이드 가능)
- **이벤트 시각화**: 가격 차트에 급등(●초록)/급락(●빨강) 이벤트 시작점 + 매매 ▲▼ 마커
- **포트폴리오**: 시드 **$10,000**, **한 번에 한 포지션**(보유 중 신규진입 금지), TP/SL로만 청산(강제 시간청산 없음)
- **자산곡선($)** vs 단순보유(Buy&Hold) 비교

## 다른 컴퓨터에서 받아 설치하기

저장소에는 소스만 들어있고 무거운 것들(`.venv`, `node_modules`, 데이터 캐시)은 제외돼 있어
받은 뒤 아래를 한 번 실행해야 합니다. **필요:** Python **3.14**, Node 22+.

```powershell
# 1) 클론
git clone https://github.com/beum1998/snd.git
cd snd

# 2) 파이썬 엔진 (Python 3.14 필수 — numba가 3.14를 지원해야 stumpy 설치됨)
py -3.14 -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 3) 웹
cd web && npm install && cd ..

# 4) 실행 (엔진 API + 웹 동시)
cd web && npm run dev
```

> 데이터(바이낸스 분봉)는 **첫 백테스트 실행 때 자동으로 내려받아** `data/`에 캐시됩니다.
> API 키 불필요. 첫 실행만 1~2분(50k 캔들 다운로드), 이후는 캐시 사용.
