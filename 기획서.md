# 크립토 패턴 트레이딩 봇 — 기획서 (v0.2)

> 상태: v1 MVP 완성·검증 완료(null) → **v1.5 전조 패턴 마이닝 구현 완료**
> 대상 시장: **크립토 (바이낸스)**
> 최종 목표: 특정 패턴을 미리 정하지 않고 **차트에서 패턴을 스스로 발견**해 매매하는 자동 봇 + 반응형 웹 UI
> 1차 목표: **패턴 엔진이 실제로 예측력이 있는지 검증하는 간단한 테스트베드**

---

## 1. 핵심 철학 (이 프로젝트에서 절대 잊지 말 것)

1. **패턴 엔진이 심장이다.** 엔진이 안 통하면 UI·실거래·크롤링은 전부 무의미. 그래서 엔진 검증이 1순위.
2. **패턴을 "찾는 것"만으로는 못 번다.** 반복되는 패턴 ≠ 이후 항상 오름. 각 패턴에 "이후 결과"를 반드시 붙여 채점한다.
3. **과최적화가 최대의 적.** 데이터가 길면 우연히 반복되는 모양이 무수히 나온다. 반드시 학습구간/검증구간을 나눠 검증한다.
4. **절대 처음부터 실제 돈 X.** 순서: 백테스트 → 테스트넷(가짜 돈, 실시간) → 소액 실거래 → 확대.
5. **수수료·슬리피지를 백테스트에 반드시 반영.** "가상은 수익, 실전은 손실"의 대표 원인.

---

## 2. 시스템 구조

```
[바이낸스]
  ├ REST API        → 과거 분봉 대량 수집 (선택된 데이터 경로)
  ├ WebSocket       → 실시간 시세 (실전 단계에서)
  └ 테스트넷        → 가짜 돈 실시간 연습
        │
        ▼
[Python 매매·분석 엔진] (24h 상시 가동)
   ├ 데이터 수집·저장 모듈 (로컬 parquet/DB 캐시)
   ├ 종목 스캐너 (급등 + 유동성 필터 → 관심목록)
   ├ 패턴 엔진: 발견(Matrix Profile) → 채점 → 선별
   ├ 백테스트 모듈
   ├ 리스크 관리 (손절/익절/포지션 크기)
   ├ 주문 실행기
   └ [확장 슬롯] 뉴스·커뮤니티 신호 (나중에 플러그인)
        │  성과·상태·체결 데이터
        ▼
[DB] ──▶ [Next.js + TypeScript 대시보드 (반응형)]
              설정 UI / 실시간 차트 / 수익률 그래프 / 패턴 시각화
```

- **프론트(Next.js/TS)와 엔진(Python)은 분리.** 엔진은 24시간 도는 백그라운드 서비스, 프론트는 그 상태를 보여주는 화면.

## 3. 패턴 엔진 (심장) — 3단계

사용자 아이디어 = 시계열 **모티프 발견(Motif Discovery)**. 며칠~몇 주치 분봉을 넣어 반복 모양을 스스로 찾고, 이후 그 모양이 재현되면 매매.

```
1. 발견   : Matrix Profile(STUMPY)로 반복되는 캔들 모양(motif) 자동 수집
             - 학습 없음. 계산만. 블랙박스 아님, 눈으로 검증 가능.
2. 채점   : 각 패턴이 과거 등장했던 모든 순간의 "이후 결과"를 통계로 매김
             - 예: 패턴 A → 이후 47번 중 39번 상승(평균 +2.1%)  ✅
             -     패턴 B → 30번 중 15승15패                     ❌ 버림
3. 선별   : 통계적으로 일관되게 수익 나는 패턴만 실전 투입
```

**설계 파라미터(추후 실험으로 튜닝):**
- 패턴 길이: 몇 개 캔들로 볼지 (짧으면 노이즈 / 길면 잘 반복 안 됨) → 여러 길이 실험
- 유사도 기준: 보통 가격대는 정규화하고 "모양"만 비교
- 입력 특징: 캔들 모양 + **거래량** 함께 보면 예측력↑

**난이도 스펙트럼(참고):** 지표룰(0) → 기하패턴(1) → **통계발견(2)** → **유사도매칭(3)** → 클러스터링(4, 가벼운 ML) → 딥러닝(5). 시작은 2~3, 확장은 나중에.

## 4. 데이터

- **형식:** OHLCV (시각 / 시가·고가·저가·종가 / 거래량) — 캔들 하나당 이 값들.
- **경로(선택됨): 바이낸스 REST API `/klines`**
  - 요청당 최대 1000개, 페이지네이션으로 몇 년치 수집.
  - 한 번 받아 **로컬(parquet/DB)에 저장 → 재사용.** 분석마다 API 재호출 안 함.
- 참고: 메이저(BTCUSDT 등)는 2017년부터 1분봉 존재. 1년치 ≈ 52만 행(수십 MB). 저장 부담 없음.
- 라이브러리: `ccxt`(거래소 연동), `pandas`(데이터 처리).
- 실시간 단계에서는 WebSocket 추가.

## 5. 종목 선정 — 2단계 깔때기

바이낸스 USDT 마켓 300+ 개를 다 분석하지 않는다.

```
[1단계 스캐너] 값싼 지표로 전 종목 훑기 (자주)
   /ticker/24hr 한 번 호출로 전 종목 24h 통계 수집
   필터: USDT페어 / 거래량 하한 / 등락·변동성 상위 /
         레버리지·스테이블 제외 / 상장기간 충분
   → 관심목록 20~30개
        │
[2단계 패턴분석] 좁혀진 후보에만 무거운 분석
   → 매매 신호
```

- **유동성 필터 필수.** 잡코인은 슬리피지·미체결·펌프앤덤프 위험.
- **개발 순서:** 초기엔 고정 소수 종목(BTC/ETH/SOL 등)으로 엔진 검증 → 안정되면 동적 스캐너 부착.

## 8. 단계별 로드맵

### ★ v1 — MVP 테스트베드 (지금 목표, 범위 최소화)
```
1. 과거 분봉 데이터 1종목 받아오기 + 로컬 저장
2. Matrix Profile로 패턴 찾기
3. "그 패턴 이후 결과" 통계 내기 (채점)
4. 백테스트로 승률·수익 확인 (학습/검증 구간 분리)
5. 결과를 간단한 화면에 표시
   ✗ 실거래 / ✗ 크롤링 / ✗ 화려한 UI / ✗ 동적 스캐너
   → 오직 "패턴 엔진이 통하나?"만 검증
```

### ★ v1.5 — 급등·급락 전조(precursor) 패턴 마이닝 (2026-07-21 구현)

**v1 결과(정직한 null):** 전 구간 반복 모양(motif)은 이후 방향이 랜덤 → 예측력 없음.
**대응:** 질문을 바꾼다 — "반복되는 모양이 있나?" → **"급등/급락에는 공통된 전조가 있나?"**

```
① 라벨링(events.py)   : 트리플 배리어 — K캔들(기본 24) 안에 ±3×ATR에 먼저 도달하면
                        급등/급락 이벤트. (반대 방향 1.5×ATR 먼저 닿으면 무효)
                        ATR 배수라 변동성 장세와 조용한 장세에서 공평.
② 전조 추출           : 이벤트 '직전' L캔들(기본 48). 이벤트 첫 캔들 절대 미포함(누수 차단).
③ 클러스터링(precursor.py): 전조들을 OHLC+로그거래량 5채널 z정규화 → k-means(방향당 6개)
                        → 클러스터 대표(medoid)가 템플릿. 크기 미달 클러스터 폐기.
④ 검증(리프트) ★심장★ : 리프트 = P(이벤트|템플릿 매칭) ÷ 기저율.
                        학습 리프트는 클러스터 멤버 자신을 반드시 제외하고 계산
                        (멤버는 전원 이벤트 전조라서 포함하면 가짜로 높아짐).
                        자동선별은 학습 기준만 사용(학습 매칭≥30 & 리프트≥1.5).
                        검증구간 리프트≥1.2 & 매칭≥15면 '검증 통과' 배지(표시용).
⑤ 거래                : TP/SL = 이벤트 정의와 동일한 ±3×ATR/∓1.5×ATR (진입 시점 ATR).
```

**고정 원칙:** 이벤트 정의(3×ATR/1.5×ATR/K=24)는 처음 정한 값으로 고정. 결과 보고 바꾸면 과최적화.

**첫 검증 결과 (BTC/USDT 5m, 50,000캔들, 2026-07-21):**
- 학습 이벤트: 급등 896 / 급락 927 (기저율 ~26/29% — 이벤트가 생각보다 흔함)
- 12개 템플릿 중 2개(P1 롱, P6 숏)가 학습 리프트 ≥1.5 통과, **검증구간에서도 리프트 재현**
  (P1: 학습 ×1.56 → 검증 ×1.75 / P6: ×1.66 → ×1.46) → 전조에 정보가 있을 가능성.
- 그러나 포트폴리오는 -49% (vs 단순보유 -10%). 원인: 5분봉 ATR 중앙값 ~0.15% →
  TP ~0.46%인데 왕복 비용이 0.30% → **움직임이 수수료에 다 먹힘.**
- 결론: 신호(리프트)는 있어 보이나 5m에서는 거래 비용을 못 이김.

**2차 실험 — 경제성 있는 스케일로 이벤트 정의 상향 (2026-07-21):**
- 원칙에서 역산해 정의 변경(스캔 아님): **TP ≥ 왕복비용×5 = 1.5%** → 10×ATR / 5×ATR / K=144(12h).
  손익분기 승률 77% → 46.7%로 개선. 이 정의를 새 고정값으로 확정.
- 결과 (BTC 5m 50k): 학습 이벤트 급등 136 / 급락 146 (의도대로 희귀해짐).
  **11개 템플릿 전부 학습 리프트 ≈ 1.0 (0.97~1.16) → 자동선별 0개. 정직한 null.**
- 해석: 작은 움직임(3×ATR≈0.46%)의 전조로 보였던 신호가 **경제성 있는 큰 움직임(1.5%)
  스케일에서는 사라짐**. 4시간짜리 캔들 모양만으로는 12시간 내 1.5% 방향을 예측 못 함.
- 다음 후보: ① 1h봉에서 3×ATR(≈1.5%, 신호가 있던 배수 그대로 + 경제성 충족 — 두 조건이
  자연히 만나는 지점) 재검증, ② 다른 종목 반복 검증, ③ 모양 외 특징(펀딩비·오더북 등)은 장기 과제.

**패턴 평가 체계 (2026-07-21 확정):**
- 선별은 가중치 합산이 아닌 **순차 게이트(AND)**: ①표본(학습 매칭≥30) → ②신호(학습 리프트≥1.5)
  → ③재현(검증 매칭≥15 & 리프트≥1.2, 배지 표시용) → ④경제성(수수료 차감 수익>0).
  이유: 가중합은 한 항목의 강점이 다른 항목의 치명상을 가림. 검증 수치는 선별에 쓰지 않음(오염 방지).
- **추천 점수**(카드 정렬 기준) = 학습 적중률의 **Wilson 95% 신뢰하한 ÷ 기저율**
  = "우연이 아니라고 95% 확신할 수 있는 최소 리프트". 표본이 적으면 자동으로 깎여서
  '리프트 높은데 표본 부족' 함정을 가중치 없이 해결. **검증 데이터는 절대 미사용.** 점수>1.0 = 신호 있음.
- UI: 패턴 카드를 점수순 정렬(#순위 표시), 카드에서 '매칭 이후 추세'(이벤트 스터디) 시각화 —
  매칭 진입 이후 K캔들 수익률 경로 스파게티 + 중앙값 + TP/SL 가이드. 중앙값이 0% 근처면 신호 없음.
- **전조 길이 다중 스케일**: 최소~최대 사이를 2배씩(기본 24→48→96캔들) 동시 탐색.
  촘촘한 전 길이 스캔은 후보 폭증 = 다중 비교 위험이라 '옥타브' 스케일만 다양화.

## 9. 주요 리스크·주의사항

- **대부분의 자동매매 봇은 돈을 잃는다.** 기대치 현실화.
- **급등 추종은 이미 늦은 경우 많음** (HFT·봇과 속도 경쟁).
- **과최적화**: 과거에만 맞고 미래엔 안 통하는 패턴 함정.
- **수수료·슬리피지** 미반영 시 백테스트 착시.
- **API 키 보안**: 출금 권한 없는 키 사용, 키 노출 금지.
